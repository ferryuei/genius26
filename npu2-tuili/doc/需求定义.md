# NPU推理加速器需求定义

## 项目概述

**项目名称**: 高性能Transformer推理专用NPU  
**实现平台**: FPGA (Intel Stratix 10)  
**应用场景**: 数据中心/边缘服务器的大语言模型推理加速  
**版本**: V2.0  
**日期**: 2026-02-27

---

## 1. 功能需求

### 1.1 核心功能
- 支持Transformer架构神经网络推理
- 重点优化Self-Attention、MLP、Layer Norm等核心算子
- 支持主流LLM模型(GPT、BERT、LLaMA等架构)
- 支持INT8量化推理,提供最高整数算力密度
- **支持BFloat16(BF16)混合精度推理**,平衡精度与性能

### 1.2 算子支持
| 算子类型 | 优先级 | 说明 |
|---------|--------|------|
| MatMul/GEMM | P0 | Transformer核心计算 |
| Softmax | P0 | Attention计算必需 |
| Layer Normalization | P0 | 标准化层 |
| GELU/ReLU | P0 | 激活函数 |
| Add/Mul | P0 | 逐元素运算 |
| Embedding | P1 | 词嵌入查找 |
| Transpose | P1 | 张量重排 |
| Slice/Concat | P2 | 张量操作 |

### 1.3 模型支持
- **序列长度**: 128 ~ 8192 tokens
- **隐藏层维度**: 512 ~ 8192
- **注意力头数**: 8 ~ 64
- **层数**: 6 ~ 96层
- **参数规模**: 支持100M ~ 70B参数模型

---

## 2. 性能需求

### 2.1 计算性能
- **目标算力**: 
  - INT8: > 20 TOPS (发挥S10最大性能)
  - BF16: > 5 TFLOPS
- **矩阵乘法效率**: > 85% DSP利用率
- **批处理大小**: 1 ~ 64
- **延迟目标**: 
  - Token生成延迟 < 30ms (7B模型, BF16)
  - 首Token延迟 < 150ms

### 2.2 带宽需求
- **DDR4带宽**: ≥ 68 GB/s (2133MHz x 4通道)
- **片上缓存**: 
  - 权重缓存: ≥ 20MB
  - 激活缓存: ≥ 10MB
- **数据复用率**: > 90%

### 2.3 扩展性
- 支持多卡级联(通过GTY/以太网)
- 支持模型并行和流水线并行
- 支持动态负载均衡

---

## 3. 接口需求

### 3.1 数据接口(必选其一)

#### 方案A: 高速串行接口 (推荐)
- **接口类型**: Intel FPGA Transceiver (17.4 Gbps)
- **接口数量**: 24 ~ 48 lanes
- **总带宽**: 100 GB/s (单向)
- **协议**: Interlaken / Custom Protocol
- **应用**: 卡间互联、与主机通信

#### 方案B: 100G以太网
- **接口数量**: 2 ~ 4 端口
- **协议**: 100GbE (Intel E-Tile)
- **总带宽**: 50 GB/s
- **应用**: 网络推理服务、分布式部署

### 3.2 存储接口
- **DDR4接口**: 4通道, 72-bit (ECC), 2133/2400 MHz
- **容量需求**: 16GB ~ 64GB
- **时序**: 符合JESD79-4标准

### 3.3 调试接口
- **JTAG**: 配置和调试
- **UART**: 控制台输出
- **ILA**: 片上逻辑分析

## 4. 硬件平台约束

### 4.1 FPGA选型: Intel Stratix 10 GX/SX

**推荐器件型号**: 1SG280LU3F50E3VG (Stratix 10 GX 2800)  
**关键资源**:
| 资源类型 | 可用量 | 预算分配 |
|---------|--------|---------|
| ALM | 933,120 | 控制逻辑 20%, PE阵列 60%, 其他 20% |
| 寄存器 | 3,732,480 | 流水线寄存器 |
| Variable Precision DSP | 5,760 | INT8/BF16乘法器阵列(核心) |
| M20K存储块 | 11,721 (20Kb) | 缓存/FIFO (228MB总容量) |
| MLAB | 116,640 (640-bit) | 小型缓冲和FIFO |
| 高速收发器 | 96 (17.4 Gbps) | 高速接口 |

**优势**:
- **Variable Precision DSP**: 原生支持INT8/INT16/FP16/BF16多精度
- **DSP密度极高**: 5760个DSP块,适合大规模矩阵运算
- **M20K存储丰富**: 228MB片上存储,大幅减少DDR访问
- **高速收发器**: 17.4 Gbps支持100+ GB/s互联带宽
- **HyperFlex架构**: 寄存器丰富,支持更高频率(可达600MHz)

### 4.2 功耗约束
- **TDP目标**: < 100W (S10 GX 2800典型功耗)
- **散热方式**: 主动风冷 + 热管
- **电源设计**: 
  - VCC: 0.9V @ 40A (核心逻辑)
  - VCCP: 1.8V @ 3A (收发器)
  - 多路SMPS + LDO

## 6. 设计约束

### 6.1 硬约束
- **不使用PCIe接口** (按需求)
- **必须使用高速收发器或以太网作为主接口**
- **必须使用DDR4作为外部存储**
- **支持INT8和BF16双精度运算**

### 6.2 软约束
- 优先优化吞吐率,其次考虑延迟
- 设计应具备可扩展性
- 保留升级到更大FPGA的能力

## 9. 风险评估

| 风险项 | 概率 | 影响 | 缓解措施 |
|-------|------|------|---------|
| 时序不满足 | 中 | 高 | HyperFlex寄存器优化,降频运行 |
| DDR带宽瓶颈 | 中 | 高 | 大容量M20K缓存,优化数据复用 |
| 资源超限 | 低 | 高 | 分时复用DSP块,精简控制逻辑 |
| 收发器调试困难 | 中 | 中 | 使用成熟IP核,预留以太网备选 |
| 功耗超标 | 低 | 中 | 智能时钟门控,局部电源管理 |
| BF16精度损失 | 低 | 中 | 混合精度策略,关键层用FP32 |
