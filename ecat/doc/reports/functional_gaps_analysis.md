## 1. RTL功能不足分析

### 1.1 CoE (CANopen over EtherCAT) 模块

#### 🟥 F1-COE-01: 缺少分段传输(Segmented Transfer)支持

**严重等级**: 🔴 P0 (严重)  
**影响范围**: CoE协议完整性和实用性  

**问题描述**:
当前RTL只实现了快速传输(Expedited Transfer)，不支持分段传输：
- ✅ 支持: SDO上传/下载 (≤4字节)
- ❌ 不支持: SDO分段上传/下载 (>4字节)

**标准要求** (ETG.1000, CiA 301):
- 对于大于4字节的数据必须使用分段传输
- 支持命令: UPLOAD_SEG_REQ (0x60), DOWNLOAD_SEG_REQ (0x00)
- 需要Toggle位管理、分段缓冲、大小端转换

**影响**:
```
无法传输的数据类型:
├── 字符串对象 (Device Name, Hardware Version)
├── 数组对象 (PDO映射表)
├── 结构体对象 (复杂数据结构)
└── 大数值对象 (>4字节)
```

**测试失败**:
- CoE-Enhanced-02: 分段上传(读取0x1008 Device Name)
- CoE-Enhanced-03: 分段下载(写入0x1009 Hardware Version)

**修复估算**:
- 工作量: 200-300行RTL代码
- 状态机: 需要新增UPLOAD_SEG/DOWNLOAD_SEG状态
- 缓冲区: 需要512字节分段缓冲
- 时间: 2-3人天

**严重等级**: 🟠 P1 (重要)  
**影响范围**: 协议兼容性和设备发现  

**问题描述**:
当前只实现了最基本的对象：
```systemverilog
已实现 (本地):
├── 0x1000: Device Type

├── 0x100A: Software Version (STRING)
├── 0x1011: Restore Default Parameters
├── 0x1014: Emergency COB-ID
├── 0x1020: Store Parameters
├── 0x1C00: Sync Manager Communication Type
├── 0x1C10-0x1C13: PDO Assignment
```

**影响**:
- 主站无法获取设备详细信息

- 不符合CiA 301最小对象字典要求

**修复估算**:
- 工作量: 150-200行RTL代码
- 需要实现字符串存储和访问
- 时间: 1-2人天

### 1.3 EoE (Ethernet over EtherCAT) 模块

**严重等级**: 🟠 P1 (重要)  
**影响范围**: 网络性能和安全性  

**问题描述**:
过滤器寄存器已实现并可配置，但在帧接收路径中未使用：

```systemverilog
// 过滤器已配置
reg [191:0] filter_mac;      // 4个MAC地址
reg [3:0]   filter_enable;
reg         filter_broadcast;
reg         filter_multicast;

// 但在接收时未应用过滤
ST_REASSEMBLE: begin
    frames_received <= frames_received + 1;  // 接收所有帧
    state <= ST_FORWARD_FRAME;
end
```

**影响**:
- 所有以太网帧都被接收和转发
- CPU负载增加
- 无法实现选择性接收
- 安全隐患

### 1.4 通用功能缺失

#### 🟥 F1-GEN-01: 缺少邮箱协议超时保护

**严重等级**: 🔴 P0 (严重)  
**影响范围**: 系统可靠性和稳定性  

**问题描述**:
所有邮箱协议模块都缺少超时机制：

```systemverilog
// 当前状态机无超时保护
ST_WAIT_PDI: begin
    if (pdi_obj_ack) begin
        // 正常处理
    end
    // 如果pdi_obj_ack永不到来？
    // 状态机将永久卡住！
end
```

**风险**:
- 外部接口故障导致系统死锁
- 需要硬件复位才能恢复
- 生产环境可靠性差

**修复建议**:
```systemverilog
reg [15:0] timeout_counter;

ST_WAIT_PDI: begin
    timeout_counter <= timeout_counter + 1;
    if (pdi_obj_ack) begin
        timeout_counter <= 16'h0;
        // 正常处理
    end else if (timeout_counter > TIMEOUT_VALUE) begin
        state <= ST_ABORT;
        error_code <= TIMEOUT_ERROR;
    end
end
```
